# 開発ルール（rules）

## 1. 破壊的変更の禁止
既存の挙動や既存ロジックを壊す変更は禁止する。

## 2. 最小差分の原則
改修は目的を達成するための最小限の差分とする。
不要なリファクタリングや構造変更は行わない。

## 3. データ構造・キー名の保護
既存のキー名 / ID / 型 / 配列構造は変更してはならない。

## 4. ドキュメント更新
- 挙動に影響がある変更を行った場合は必ず `memory/changelog.md` に追記する  
- 必要に応じて spec や rules も更新する  

## 5. AI利用時の注意
改修を開始する前に必ず `memory/spec.md` と `memory/rules.md` を読み、
前提を理解してからコードを生成すること。
